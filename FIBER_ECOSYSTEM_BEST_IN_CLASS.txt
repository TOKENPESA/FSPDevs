WHAT "BEST-IN-CLASS" MEANS FOR A FIBER MESH CONTROL PLANE
================================================================================

A best-in-class Fiber mesh solution is NOT just a wrapper around fnn.exe.
It is the operational layer the ecosystem lacks today:

  1. FLEET ORCHESTRATION
     Run N Fiber nodes + sidecars with deterministic identity, ports, and
     health — from 1 edge node to 1024 simulated agents on one operator machine
     or distributed across K8s/bare metal.

  2. UNIFIED OBSERVABILITY
     Single pane: channel liquidity, payment flows, heal events, routing paths,
     CKB funding status — real-time WebSocket + historical query.

  3. INTELLIGENT ROUTING LAYER
     Control plane that understands topology AND live channel capacity;
     recommends paths; triggers liquidity moves before payments fail.

  4. PRODUCTION SECURITY
     Signed agent telemetry, mTLS or token auth, no dev keys in prod,
     audit trail for channel open/fund/payment operations.

  5. TESTNET → MAINNET PARITY
     Same APIs and dashboards against sim, local FNN, fiber.world peers.

FSPDevs CURRENT POSITION (v5 post-refactor)
================================================================================

STRENGTHS (differentiators today):
  ✓ 1024-node synthetic mesh with ring/skip/chord — unique scale story
  ✓ MFA supervisor: heal, route, hub liquidity, signed telemetry
  ✓ mesh-core shared library — clean split for ecosystem SDK potential
  ✓ mesh-fleet binary — one process, many FAs (ops innovation)
  ✓ Dashboard: live mesh viz, payments UI, liquidity events, edge-node slider
  ✓ Sidecar SQLite: offline telemetry, channel cache, tx ledger
  ✓ Local FNN bridge: list_channels, open_channel, keysend payments

WEAKNESSES (vs best-in-class):
  ✗ MFA graph not synced from real FNN/gossip
  ✗ Payments: single-hop keysend only — no multi-hop via fiber-lib router
  ✗ No funded public testnet channels documented end-to-end
  ✗ Dev deterministic keys — no HSM/vault integration
  ✗ No metrics (Prometheus), tracing (OpenTelemetry), or alert routing
  ✗ No HA: MFA single process, in-memory graph
  ✗ No public API docs / OpenAPI for third-party integrators
  ✗ Wasm path deprecated without replacement for browser-only demos

ESTIMATED ECOSYSTEM READINESS: ~50% (control plane strong, settlement weak)

ANTI-PATTERNS (do not ship to testnet)
================================================================================

  • Treating MFA synthetic graph as source of truth for real payments
  • sim-peer-* or Math.random() health in any prod code path
  • Silent broadcast/UI event drops (fixed in v5 — verify no regressions)
  • Single shared FNN hub for 1024 agents without channel isolation plan
  • keysend-only payments presented as "full Fiber routing"
