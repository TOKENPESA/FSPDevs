FIBER NETWORK TESTNET — OFFICIAL REQUIREMENTS (REFERENCE)
================================================================================

Source: https://www.fiber.world/docs/quick-start/run-a-node
        https://github.com/nervosnetwork/fiber

To run on Fiber Testnet, a team must:

  1. Deploy FNN (Fiber Network Node) binary
     - Download from GitHub releases OR cargo build --release
     - Copy config/testnet/config.yml
     - Create data directory with ckb/key (private key via ckb-cli)

  2. Fund testnet account
     - CKB faucet for native CKB
     - RUSD via JoyID wallet faucet (for UDT channels)

  3. Start node
     - FIBER_SECRET_KEY_PASSWORD env var (encrypt wallet)
     - RUST_LOG=info ./fnn -c config.yml -d .

  4. Connect to public testnet peers
     - open_channel with public nodes (min ~438-500 CKB)
     - Wait for CHANNEL_READY state
     - send_payment for actual transfers

  5. Version compatibility
     - Close channels before FNN upgrades (storage format changes)
     - Use fnn-migrate for DB upgrades

TPXDevs currently implements NONE of steps 1-4. Our MFA is a parallel
simulation layer, not an FNN wrapper or plugin.
