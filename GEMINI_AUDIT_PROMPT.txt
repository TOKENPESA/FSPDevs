# TPXDevs — Gemini Security & Code Audit Request

## Project summary
FiberAgent swarm network simulation with 3 components:

1. **master-fiber-agent** (Rust/Axum) — Supervisor on 127.0.0.1:1025
   - HTTP: POST /telemetry, POST /route, GET /
   - WebSocket: /ws/:agent_id (config push), /ws/monitor (dashboard)
   - 1024-node mesh graph (ring + skip + chord edges)
   - Mesh healing: isolate dead nodes, hot-swap peers via WebSocket

2. **fiber-agent** (Rust → WebAssembly) — Edge agents in browser
   - Heartbeat loop, MFA escalation on peer failure
   - WebSocket receive MESH_CHANNEL_HOT_SWAP commands
   - Localhost-only HTTP/WS endpoints enforced

3. **Dashboard** (vanilla JS + HTML5 Canvas) — Real-time visualizer on :8088
   - WebSocket monitor from MFA
   - 32×32 grid, 1024 nodes, click-to-toggle offline (visual only)

## Architecture flow
```
FiberAgent (Wasm) --HTTP POST--> MFA /telemetry (ALERT_MFA_NODE_DROPPED)
FiberAgent (Wasm) <--WS-------- MFA /ws/:id (MESH_CHANNEL_HOT_SWAP)
Dashboard         <--WS-------- MFA /ws/monitor (MESH_HEAL events)
```

## Audit scope — please review for:
- Security vulnerabilities (injection, CORS abuse, WS hijacking, DoS)
- Backdoors, suspicious network calls, credential leaks
- Logic bugs (healing, routing, race conditions, state leaks)
- Memory/performance issues
- Missing auth on localhost services (acceptable for dev sim?)
- Wasm-specific issues (closure leaks, unsafe patterns)
