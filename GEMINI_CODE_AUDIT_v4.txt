================================================================================

TPXDevs is a 1024-node FiberAgent mesh **simulation** with 4 runtime tiers:

TIER 1 — Fiber Network Node (FNN v0.8.1 release binary)
  Path: fnn-testnet/ (local runtime, not in audit zip)
  Bind: RPC 127.0.0.1:8227 · P2P :8228
  Role: Official FNN on CKB testnet config · 0 channels until funded

TIER 2 — master-fiber-agent (Rust / Axum / Tokio)
  Path: master-fiber-agent/src/main.rs (~646 lines)
  Bind: 127.0.0.1:1025
  Role: Master Fiber Agent (MFA) — mesh supervisor brain
  APIs:
    GET  /              — health JSON
    POST /telemetry     — MESH_HEARTBEAT + ALERT_MFA_NODE_DROPPED
    POST /route         — Dijkstra multi-hop routing over live mesh graph
    WS   /ws/:agent_id  — push MESH_CHANNEL_HOT_SWAP config to agents
    WS   /ws/monitor    — broadcast MESH_HEAL + MESH_HEARTBEAT to dashboard

TIER 3a — fiber-agent native sidecar (fiber-agent-daemon)
  Paths: fiber-agent/src/main.rs (~223), fiber-agent/src/fnn_client.rs (~178)
  Role: Poll FNN list_channels every 3s · POST MESH_HEARTBEAT to MFA
        · WS reconnect loop with ping/pong keepalive to MFA
  Default: AGENT_ID=44 · FNN_RPC_URL=http://127.0.0.1:8227
  FNN_MODE=simulate → SimulatedFnnClient fallback (no fnn.exe)

TIER 3b — fiber-agent Wasm (browser agent)
  Path: fiber-agent/wasm/src/lib.rs (~305 lines)
  Role: Edge FiberAgent — Math.random() peer health · ALERT telemetry only
  Runs in browser via fiber-agent/demo/ on :8787

TIER 4 — Dashboard + Demo (vanilla JS / HTML5 Canvas)
  Paths: dashboard.js (~615 lines), index.html
  Dashboard: :8088 — 32×32 lattice visualizer, WS monitor, route simulator
  Demo: :8787/demo/ — single Wasm agent harness

================================================================================
MESH TOPOLOGY
================================================================================

1024 agents (FA-1..FA-1024). Each node starts with 3 edges (ring/skip/chord),
stored in a **flexible Vec<MeshEdge>** per node (not a fixed [MeshEdge; 3]).

Edge formulas (N=1024, node i):
  ring  = (i == N) ? 1 : i+1
  skip  = (i >= N-1) ? 1 : i+2
  chord = (i + N/2) % N + 1

MeshEdge fields:
  target_node: u16
  capacity_shannons: u64   — per-hop liquidity gate in Dijkstra
  is_active: bool

Healing flow (Wasm or sidecar ALERT path):
  1. Agent detects peer failure
  2. Agent POSTs ALERT_MFA_NODE_DROPPED to /telemetry
  3. MFA background worker: isolate_faulty_node → pick_healing_peer
  4. MFA sends MESH_CHANNEL_HOT_SWAP via /ws/:agent_id (try_send)
  5. Agent swaps peer_id on broken channel, returns to Active
  6. MFA broadcasts MESH_HEAL to /ws/monitor → dashboard renders

Sidecar health flow (NEW in v4.1):
  1. Sidecar polls FNN list_channels via JSON-RPC every 3s
  2. POSTs MESH_HEARTBEAT to /telemetry (channels healthy)
  3. MFA apply_heartbeat_liveness syncs reported neighbors in mesh graph
  4. MFA logs + broadcasts MESH_HEARTBEAT to /ws/monitor → dashboard
  5. On FNN failure: ALERT_MFA_NODE_DROPPED → heal → WS hot-swap → sidecar open_channel

NOTE: MFA graph is NOT synced from FNN channels — routing uses in-memory sim graph.
