Mesh scale: up to 1024 agents (FA-1..FA-1024) via mesh-fleet binary.
Routing: MFA in-memory graph; NOT fully synced from live FNN channel state.
Payments: MFA dispatches MESH_SEND_PAYMENT → sidecar keysend (single-hop/direct).
Healing: isolate → pick_healing_peer → WS hot-swap → hub open_channel RPC.
Funding: ALERT_BALANCE_DEPLETED + heal fallback → trigger_hub_liquidity_provisioning.

KNOWN LIMITATIONS (TEAM-ASSERTED — VALIDATE)
================================================================================

  • Sim/live payments are keysend on direct channels only — NO multi-hop settlement
  • MFA graph is synthetic topology, not live FNN gossip sync
  • Hub open_channel params may not match all FNN v0.8.x variants
  • Dev secp256k1 keys in mesh-pubkeys — not production key management
  • MFA in-memory state — no crash recovery / cluster HA
  • Single-machine fleet — no K8s/service mesh deployment story yet
  • Mobile Money Float Bridge: Tauri + clearing_client wired; NOT in fiber-agent-daemon loop
  • Float crisis → MFA POST /clearing/float-crisis wired from Tauri rebalance sim
  • RegionalClearinghouseEngine: intent swap + telco B2C (needs MFA_TELCO_CLEARING_API_URL)
  • Fee engine: 3-tier L1/L2/L3; hop count + FX rate still hardcoded mocks in fiat_bridge
  • mTLS supervisor optional (MFA_MTLS=true + certs/); not default
  • Prometheus /metrics exposed; custom business metrics minimal
