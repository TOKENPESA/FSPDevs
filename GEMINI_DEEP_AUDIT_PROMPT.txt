         |

    Dijkstra routing · per-hop capacity_shannons
    Signed telemetry · healing · hub liquidity · sim payments API
         |
         |  WS /ws/:agent_id  (MESH_CHANNEL_HOT_SWAP, MESH_SEND_PAYMENT)
         v
  [Sidecar fleet — fiber-agent mesh-fleet or fiber-agent-daemon]
    Per-FA FNN RPC (sim or live ports) · SQLite persistence
    MESH_HEARTBEAT · ALERT_BALANCE_DEPLETED · keysend payments
         |
         |  JSON-RPC list_channels, open_channel, send_payment (keysend)
         v
  [FNN per agent or shared hub — fnn-testnet/ locally]
    CKB testnet config · channels · P2P

  [mesh-core] — shared topology, pubkeys, registry (both MFA + sidecar)

Mesh scale: up to 1024 agents (FA-1..FA-1024) via mesh-fleet binary.
Routing: MFA in-memory graph; NOT fully synced from live FNN channel state.
Payments: MFA dispatches MESH_SEND_PAYMENT → sidecar keysend (single-hop/direct).
Healing: isolate → pick_healing_peer → WS hot-swap → hub open_channel RPC.
Funding: ALERT_BALANCE_DEPLETED + heal fallback → trigger_hub_liquidity_provisioning.
