PROJECT OVERVIEW
================================================================================

TPXDevs is a 1024-node FiberAgent mesh network simulation with 3 tiers:

TIER 1 — master-fiber-agent (Rust / Axum / Tokio)
  Path: master-fiber-agent/src/main.rs (~538 lines)
  Bind: 127.0.0.1:1025
  Role: Master Fiber Agent (MFA) — mesh supervisor brain
  APIs:
    GET  /              — health JSON
    POST /telemetry     — ingest ALERT_MFA_NODE_DROPPED from Wasm agents
    POST /route         — Dijkstra multi-hop routing over live mesh graph
    WS   /ws/:agent_id  — push MESH_CHANNEL_HOT_SWAP config to agents
    WS   /ws/monitor    — broadcast MESH_HEAL events to dashboard

TIER 2 — fiber-agent (Rust → WebAssembly via wasm-bindgen)
  Path: fiber-agent/src/lib.rs (~278 lines)
  Role: Edge FiberAgent — heartbeat loop, MFA escalation, WS hot-swap receiver
  Runs in browser via fiber-agent/demo/

TIER 3 — Dashboard + Demo (vanilla JS / HTML5 Canvas)
  Paths: dashboard.js (~501 lines), index.html, fiber-agent/demo/main.js
  Dashboard: :8088 — 32×32 lattice visualizer, WS monitor, click-to-offline (visual only)
  Demo: :8787/demo/ — single Wasm agent harness

================================================================================
MESH TOPOLOGY (CRITICAL CONTEXT)
================================================================================

1024 agents (FA-1..FA-1024), each with exactly 3 mesh edges:
  • Ring  — successor i+1 (wraps at 1024)
  • Skip  — i+2 (wraps)
  • Chord — opposite pole i+512 (wraps)

Mesh healing flow:
  1. Wasm agent detects peer failure (max_failures consecutive misses)
  2. Agent POSTs ALERT_MFA_NODE_DROPPED to /telemetry
  3. MFA background worker: isolate_faulty_node → pick_healing_peer
  4. MFA sends MESH_CHANNEL_HOT_SWAP via /ws/:agent_id
  5. Agent swaps peer_id on broken channel, returns to Active
  6. MFA broadcasts MESH_HEAL to /ws/monitor → dashboard renders
