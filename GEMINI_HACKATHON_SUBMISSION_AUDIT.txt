----- FIELD 1: Submission category -----

Category 2: Node, Routing, Cross-Chain, and Diagnostics Infrastructure


----- FIELD 2: Project overview -----

TOKENPESA / FSPDevs Fiber Agent is a device-shippable Fiber Network edge stack:
a Windows desktop Sidecar (Tauri) that auto-registers to an MFA fleet control
plane, bundles a Fiber Network Node (FNN), and gives field operators a console
for channels, Fiber testnet payments, DiCoBa savings modules, and diagnostics.

Target audience: retail/field agents, merchant operators, and infra operators
who need a installable Fiber edge node + MFA-supervised fleet identity — not
raw FNN CLI alone.


----- FIELD 3: What problem does it solve? -----

Fiber Network participation usually requires juggling: node binaries, CKB L1 RPC,
channel funding, agent identity, and operator UX — easy to leave L1 “demo”
bookkeeping that looks like settlement.

FSPDevs solves the edge deployment gap for Fiber infrastructure by:
  • Packaging FNN inside the Windows installer (Tauri externalBin)
  • Binding Fiber JSON-RPC strictly to localhost (:8227) while verifying L1 via
    the Treasury Hub CKB façade (:8114)
  • Issuing MFA FA-N identities and keeping signed telemetry / control WSS live
  • Routing SQLite + node state to %APPDATA%\TokenPesa\state (installer-safe)
  • Hard-failing when live FNN is unreachable (no silent SimulatedFnnClient on
    testnet mode)

Relation to Fiber Network: we are an edge+hub Fiber node fleet with routing/
payment execution over FNN RPC and Fiber P2P to the Treasury Hub, plus operator
diagnostics — not a separate L1 token project.


----- FIELD 4: System design -----

Topology:
  [Windows Sidecar FA-N + bundled FNN :8227]
           | HTTPS / WSS (HMAC)
           v
  [MFA mfa.fsprotocol.com — register, control, telemetry]
           ^
           | Fiber P2P / channel mesh
  [Treasury Hub FA-1 + Linux FNN :8228]
           |
           v
  [CKB testnet L1 via Hub :8114 → testnet.ckbapp.dev]

Important USER flows:
  1. Install NSIS → app boots FNN → MFA auto-register → FA-N visible on MFA
  2. Fund address (JoyID/faucet UI) → open/monitor channels → send_payment
  3. Operate DiCoBa / cash / OOB QR modules when mounted
  4. If FNN fails port 8227 → fatal gate (truthful live mode)

Important DEVELOPER / OPS flows:
  1. npm run prepare:fnn-sidecar → npm run tauri:build → NSIS artifact
  2. Override via FIBER_AGENT_STATE_DIR, CKB_NODE_RPC_URL, FNN_MODE=simulate
     (simulate is explicit demo only)
  3. Hub: deploy/droplet scripts — FNN systemd + nginx CKB/FNN RPC façades
  4. Probe L1: POST get_tip_block_number to http://134.122.120.65:8114


----- FIELD 5: Setup environment -----

Local / ship environment (Windows edge):
  • OS: Windows 10/11 x64
  • Rust stable + MSVC (x86_64-pc-windows-msvc)
  • Node.js + npm (@tauri-apps/cli v2)
  • WebView2 (Tauri desktop)
  • Bundled FNN Windows release binary
  • SQLite (rusqlite) under %APPDATA%\TokenPesa\state
  • Defaults: FNN_MODE=testnet, MFA_HOST=mfa.fsprotocol.com (TLS)

Hub / MFA environment (Linux droplets):
  • Ubuntu/Debian, systemd, nginx
  • Linux FNN + fiber-agent-daemon (Hub)
  • master_fiber_agent (MFA)
  • CKB JSON-RPC reverse proxy (not a full local CKB sync node)


----- FIELD 6: Tooling -----

Fiber / CKB:
  • Fiber Network Node (fnn) v0.8.x — JSON-RPC client LiveFnnClient
    (node_info, list_channels, open_channel, send_payment, …)

    (cell deps from nervosnetwork/fiber-scripts testnet migrations)
  • CKB JSON-RPC — get_tip_block_number and related via Hub proxy or
    https://testnet.ckbapp.dev/
  • JoyID web SDK / faucet integrations inside Sidecar WebView CSP allowlist

App / ops tooling:
  • Tauri 2 (externalBin sidecar packaging, shell plugin spawn)
  • Rust crates: tokio, axum (module API), secp256k1, rusqlite, dirs, reqwest
  • PowerShell: fnn-testnet/*, prepare-fnn-sidecar.ps1, droplet remote bootstrap
  • No custom on-chain contract compiler SDK beyond Fiber’s published scripts


----- FIELD 7: Current functionality -----

Windows device ship:
  • NSIS installer “Fiber Agent_1.0.0_x64-setup.exe” with bundled FNN
  • On boot: spawn FNN with --ckb-node-rpc-url Hub:8114 and
    --rpc-listening-addr 127.0.0.1:8227; data under TokenPesa\state\fnn
  • MFA auto-registration when no token; persist FA-N + secret; control WSS
  • resolve_fnn_backend: ping live node or panic / fatal UI (simulate only if
    FNN_MODE=simulate|sim)

Fiber operations (via LiveFnnClient + console):

  • Operator UI: dashboard, funding (JoyID), DiCoBa, cash desk, OOB QR, module

Infrastructure already live:
