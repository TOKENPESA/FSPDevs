1.4 Why nginx for FNN RPC

  FNN refuses binding JSON-RPC on a public address without biscuit_public_key.
  This deployment does NOT use Biscuit. Pattern:
    FNN listens 127.0.0.1:18227
    nginx listens 0.0.0.0:8227, allow 167.99.150.153 + 127.0.0.1, deny all
    MFA HUB_RPC_URL=http://134.122.120.65:8227
    FA FNN_RPC_URL=http://127.0.0.1:18227
  Scripts: deploy/droplet/nginx-fnn-rpc.sh, fnn-testnet-hub.config.yml

3. MFA IMPLEMENTATION DETAIL
================================================================================

3.1 Entrypoint

  master-fiber-agent/src/main.rs → master_fiber_agent::run() in lib.rs
  Boot: tokens, clearing env check, telemetry mpsc (TELEMETRY_QUEUE≈8192),
  graph snapshot hydrate, SQLite module store, background workers, Axum bind
  (optional mTLS via MFA_MTLS).

3.2 Bind / env (deploy-relevant)

  MFA_BIND_ADDR          (e.g. 127.0.0.1:1025 behind nginx)
  MFA_API_TOKEN          Bearer / X-MFA-API-Token (min 16 chars in release)
  MFA_AGENT_WS_TOKEN     Agent WS shared secret
  MFA_WS_ALLOWED_ORIGINS Exact Origin allowlist (+ helpers)
  HUB_RPC_URL / FNN_RPC_URL → EnterpriseFnnClient (treasury clearing/refuel)
  MESH_PUBKEY_REGISTRY_PATH / MESH_PUBKEY_REGISTRY
  HUB_FUNDING_SHANNONS, HUB_FUNDING_LOCK_TIMEOUT_SECS
  MFA_* money rate envs, clearing/PAPSS/mTLS optionals
  Example: deploy/droplet/env.mfa.example
  Live env path pattern: /etc/fspdevs/mfa.env

3.3 HTTP / WS surface

Public (no API token):
  GET  /                 health — exposes connected agents, auth hints, hub metadata
  GET  /simulation
  GET  /ws/:agent_id     agent control plane (token/HMAC + Origin)

Protected (require_mfa_api_auth):
  POST /telemetry
  POST /route
  POST /simulation
  POST /clearing/float-crisis | /clearing/b2b-remittance | /clearing/multi-asset
  POST /gossip/channel
  Compliance ticket + SSE streams (+ /api/v1/compliance/*)
  GET  /ws/monitor       UI event bus
  /api/modules/*         plugin catalog / install / toggle
  GET  /metrics          Prometheus

Handlers under master-fiber-agent/src/handlers/

3.4 Auth model

  API token: Bearer, X-MFA-API-Token; ?token= rejected on non-WS HTTP after match.
  Agent WS: preferred HMAC-SHA256 headers
    X-MFA-Agent-Auth, X-Agent-ID, X-MFA-Timestamp (±300s)
    message = "{agent_id}:{timestamp}"
  Legacy: ?token= query (constant-time compare) — **current FA daemon uses this**.
  Telemetry: API token + secp256k1 signature + pubkey registry binding +
    ±15s timestamp skew + monotonic nonce high-water mark (IN-MEMORY).
  Compliance streams: 30s single-use tickets.
  Optional mTLS: MFA_MTLS=1 + CA/client certs.
  CORS / CSWSH: exact Origin allowlist.

3.5 Core modules (audit paths)

  auth.rs, middleware.rs, config.rs
  telemetry.rs          signature + nonce HWM
  graph.rs, routing.rs, micro_routing.rs
  payment.rs            multi-hop WS payment engine

3.6 Persistence (MFA)

  SQLite supervisor DB: MFA_SUPERVISOR_DB_PATH or .mfa-supervisor/mfa-supervisor.db

4. FIBER AGENT / TREASURY HUB (FA-1) IMPLEMENTATION DETAIL
================================================================================

4.1 Entrypoint

  fiber-agent/src/main.rs → run_agent_sidecar (daemon.rs)
  Does NOT spawn module API in daemon path (spawn_module_api_server is Tauri).
  Deploy healthchecks / nginx may still assume :19444 — SOURCE vs DEPLOY MISMATCH.

4.2 Sidecar loop (daemon.rs)

  • Connect MFA control WS: ws://{MFA_HOST}/ws/{AGENT_ID}?token=…
    Origin header hardcoded to http://127.0.0.1:8088 (must stay in MFA allowlist)
  • Poll / emit signed telemetry → MFA /telemetry (API token + signature)
  • Offline SQLite queue offline_telemetry_queue with retry
  • Optional FNN client (live JSON-RPC or FNN_MODE=simulate)
  • Utility / float-crisis paths → MFA clearing endpoints

4.3 Bind / env (Treasury)

  AGENT_ID=1
  MFA_HOST=<MFA IP or hostname>
  MFA_AGENT_WS_TOKEN, MFA_API_TOKEN (telemetry)
  FNN_RPC_URL=http://127.0.0.1:18227   (live; FNN_MODE unset)
  FSP_AGENT_SECRET_KEY / FIBER_AGENT_SECRET_KEY_HEX
  FIBER_AGENT_ALLOW_DEV_KEYS=false (required on Hub)
  FIBER_AGENT_STATE_DIR / FIBER_AGENT_DB_PATH
  MESH_PUBKEY_REGISTRY_PATH
  Example: deploy/droplet/env.treasury.example
  Live: /etc/fspdevs/treasury.env

4.4 Storage (FA)

  SQLite fa-{id}.db under FIBER_AGENT_STATE_DIR (live often /root/.fiber-agent/fa-1.db)
  Tables include channels, offline_telemetry_queue, ledgers, modules, etc.
  sanitize_storage_error() — never surface raw rusqlite to clients.

4.5 Module API gap

  fiber-agent/src/api/mod.rs — binds FIBER_AGENT_BIND_ADDR (default 127.0.0.1:19444)
  Routes /api/modules/* with NO auth middleware found.
  Called from Tauri boot; NOT from fiber-agent-daemon main.
  If nginx/treasury host conf proxies :80 → :19444 while daemon lacks listener → 502.
  If exposed without auth → install/uninstall risk. HIGH PRIORITY FOR GEMINI.

4.6 Key FA modules

  identity.rs, fnn_client.rs, payment.rs, hot_swap.rs
  storage.rs (AsyncDbQueue pattern for high-velocity WAL)
  module_host.rs, modules/* (fiat bridge, DICOBA, …)
  clearing_client.rs
  profiles: fiber-agent/profiles/{full,coop,kiosk,relay}.profile.toml

9. KNOWN GAPS / QUESTIONS FOR GEMINI
================================================================================

P0 candidates
  1. TLS before any public pilot; HSTS/certs.
  2. Rotate any tokens committed in provision scripts; confirm live secrets uniqueness.
  3. Close FA module API: bind localhost-only AND add auth OR remove nginx proxy.
  4. Move FA agent WS to HMAC headers; stop putting tokens in query strings/logs.
  5. Persist MFA telemetry nonce HWM (or sequence store) across restarts.

P1 candidates
  6. Align CSP / MFA_WS_ALLOWED_ORIGINS / FA Origin header with droplet URLs.
  7. Resolve daemon vs :19444 healthcheck/nginx mismatch.
  8. Application rate limits on /telemetry, /route, /clearing, /api/modules.
  9. Fund Hub CKB address; open/verify at least one Fiber channel; end-to-end payment test.
 10. Document FNN wallet backup + rotation; consider Biscuit if RPC must leave allowlist.

P2 candidates
 11. Public health endpoint information disclosure minimization.
 12. Eliminate remaining float money paths (DICOBA) in favor of fixed-point.
 13. Dual identity clarity: agent signing pubkey vs Fiber node pubkey in registry/UI.
 14. Observability: structured logs + metrics SLOs for WS disconnects / 409 replay.
 15. Private networking / VPC between MFA and Hub if DO supports it.

Explicit unknowns
  • Whether live Hub nginx currently proxies :80 → :19444 while API is down.
  • Completeness of mesh-pubkeys vs all future edge AGENT_IDs.
  • PAPSS gateway enabled or mocked in live MFA.
  • Certificate/mTLS usage (likely off).
