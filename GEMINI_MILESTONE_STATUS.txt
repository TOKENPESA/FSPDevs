MILESTONE 1 — MFA APP STORE BACKEND
================================================================================

Problem solved:
  MFA policy plugins could not be installed/uninstalled from browser UI.
  Catalog showed "Loading…" / "Failed to fetch" (CORS).
  Uninstalled plugins reappeared after restart (bootstrap re-seed bug).

Implementation:
  master-fiber-agent/src/mfa_storage.rs
    - installed_modules + mfa_registry_meta tables
    - plugin_registry_bootstrapped flag (persisted)
    - hydrate does NOT re-seed after user uninstalls all modules

  master-fiber-agent/src/policies/registry.rs (PluginHotReloader)
    - hydrate_from_storage(), install_and_mount(), uninstall(), toggle()
    - mark_plugin_registry_bootstrapped() on user install/uninstall

  master-fiber-agent/src/api/plugin_routes.rs
    GET  /api/modules/catalog
    GET  /api/modules/installed
    POST /api/modules/install
    POST /api/modules/uninstall
    PUT  /api/modules/toggle

  master-fiber-agent/src/auth.rs + config.rs
    - is_allowed_cors_origin() shares allowlist with WebSocket origins
    - PUT added to CORS methods
    - Origins: :8088 (legacy console), :5173 (React console), loopback variants
    - MFA_DEV_MODE=true allows any loopback port in dev

  master-fiber-agent/src/handlers/health.rs
    - running_plugins in health JSON

  master-fiber-agent/src/storage_error.rs (NEW)
    - sanitize_storage_error() — raw rusqlite text never returned to HTTP clients

  fnn-testnet/start-live-mfa.ps1
    - Stop-ExistingMfa before bind (port 1025 conflict fix)
    - MFA_DEV_MODE=true for local console dev
